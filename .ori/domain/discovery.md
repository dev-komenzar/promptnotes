# Discovery {#discovery}

PromptNotes は **AI プロンプトを書き溜め、必要な時にすぐコピーして使う** ための
ローカルファースト・デスクトップノートアプリ（Tauri v2）。
本文書は Phase 1 として、その事業文脈を言語化する。

## Problem Space {#problem-space}

AI を業務に組み込む developer / engineer は、効果的なプロンプトを
**何度も再利用・改変しながら使う** が、既存ツール群に以下の不満がある。

- 汎用ノートアプリ（Notion / Obsidian / Apple Notes 等）は
  「**思いついたプロンプトを 1 秒で起案して、1 秒で取り出してコピーする**」動線が弱い
- プレビュー↔編集モードで別レンダリングエンジンを使うエディタは
  Markdown 編集中に**カーソル位置・行高がズレ**、プロンプト推敲の集中を妨げる
- サイドバー・アプリ名ヘッダーといった**装飾 UI が本文表示領域を侵食**する
- クラウド同期型のサービスはプロンプトという**個人資産をベンダーに預ける**形になり、
  そのまま他ツール（CLI スクリプト / 別 AI クライアント）への持ち出しが面倒
- **複数デバイスでの作業**：デスクトップ PC とノート PC の両方でプロンプトを
  使いたいが、クラウドアカウントを作りたくない。Syncthing 等の P2P ファイル同期
  ツールと `.md` ファイルベースの相性は良いが、**既存アプリは外部からのファイル変更を
  検知できず、同期先デバイスでの編集が現在のデバイスに反映されない**

これら不満は spec 末尾 "過去の実装で AI が犯したミス" として禁止事項化されている
（サイドバー追加 / アプリ名ヘッダー追加 / レンダラ切替）。
PromptNotes はこれらを意図的に排除した形で課題を解く。

## Core Domain {#core-domain}

このプロダクトでなければ実現できない中核領域は次の 4 点に集中する。

1. **プロンプト・ブロックのストック**
   - 1 ノート = 1 つの `.md` ファイル（frontmatter + 本文）
   - フィード最上部の入力欄からワンキー（`Cmd+N` → `Cmd+Enter`）で起案・保存
   - Logseq ジャーナルビュー型の「ブロックが連続するドキュメント」を採用し、
     カード型 UI を**意図的に**避ける

2. **座標ズレのない Markdown 編集体験**
   - **CodeMirror 6 を `readOnly` トグルで使い回す**ことで
     プレビュー時と編集時のレンダラを統一する
   - これは spec で禁止事項として明示されており、Core の競合差別化点でもある

3. **本文のみのクリップボード出力**
   - ホバー時のコピー操作で frontmatter / タグを除外し本文だけを返す
   - 「書く」と「使う（外部 AI に貼る）」の間の摩擦を最小化する

4. **外部ファイル変更の検知と反映**
   - `storage_dir` 配下の `.md` ファイルが vim / VSCode / Syncthing 等の
     外部プログラムによって変更されたことを OS レベルで検知する
   - 変更検知時はノート一覧・エディタ内容を**自動的に最新化**する
   - 編集中ノートと外部変更が競合した場合は**ユーザに選択肢を提示**し、
     「外部変更を適用」「編集中を保持」を選ばせる
   - クラウドやアカウントに依存せず、`.md` ファイルのローカル所有性を
     保ったまま複数デバイス・複数ツールとの併用を可能にする

それ以外（タグ編集 UI、設定モーダル、自動アップデート等）は supporting subdomain
として扱う。AI 呼び出し機能やクラウド同期機能は Non-Goal。

## Business Drivers {#business-drivers}

成功指標として以下を重視する。

- **時間短縮**：プロンプトを「起案 → 検索 → コピーして外部 AI に貼る」までの
  動線時間を最小化する（Cmd+N / Cmd+F / ホバー → コピー で完結）
- **ストック場所の提供**：プロンプトを書き溜める**ローカル所有の置き場**そのものを
  提供する（クラウド依存・アカウント不要・`.md` で持ち出し自由）
- **マルチデバイス対応**：Syncthing 等の P2P ファイル同期ツールと組み合わせることで、
  デスクトップ PC とノート PC 間でプロンプトをシームレスに共有できる。
  PromptNotes 自身は同期プロトコルを実装せず、**ファイルシステム上の変更を検知して
  即座に UI へ反映する**ことでこれを実現する

これらは互いに補完関係にあり、ローカル `.md` 保存（所有性）が外部ツールとの
互換性も同時に提供する（後者は driver というより Core Domain の帰結）。

## Non-Goals {#non-goals}

意図的にやらないことを境界として明示する。

- **AI API 呼び出し / 推論機能**：書いたプロンプトはコピーして外部に渡す前提
- **クラウド同期 / アカウント / 共有機能**：クラウドバックエンド、ユーザ登録、
  共有リンク、リアルタイムコラボレーション等は対象外。
  ただし **ローカルファイルシステム上の変更検知**（外部プログラムによる
  `.md` ファイルの変更を OS レベルで検出し UI に反映すること）は Core Domain に含む
- **サイドバー UI**：レイアウトはシングルペインのみ
- **アプリ名ヘッダー**：OS ネイティブのタイトルバーで代替
- **プレビュー専用レンダラとの切替**：CodeMirror の `readOnly` トグルのみ
- **本文の折りたたみ / 切り捨て**：ブロックは常に全文表示

これらは spec で明示された制約であり、Phase 9 以降の workflow / UI 設計で
逸脱が起きた場合は **discovery を変えるのではなく実装を直す**。

## Stakeholders {#stakeholders}

- **主ユーザ：AI を使う developer / engineer**
  - 関心：プロンプト推敲の速度、所有性、他ツール（CLI / 別 AI クライアント）との
    `.md` ベース連携
  - 利用文脈：日次でプロンプトを起案・改変し、頻繁に外部 AI に貼り付ける

- **プロダクト開発者（dev-komenzar）**
  - 関心：シングルペイン原則 / CodeMirror 一貫性 / OS ネイティブ統合
  - 過去の実装ミス（spec 末尾）を**再発させない** gatekeeper としても機能する

- **OS（macOS / Linux / Windows）**
  - 関心：ネイティブ titlebar、`Cmd` / `Ctrl` キーバインド、OS 慣習の保存パス、
    ゴミ箱への移動 API
  - PromptNotes はこれらを**尊重する** stakeholder として扱う
